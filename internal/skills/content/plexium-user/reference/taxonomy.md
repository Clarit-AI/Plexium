# Page Taxonomy

Wiki pages are organized into sections (subdirectories under `.wiki/`). When creating a new page, choose the right section based on what the page documents.

## Sections

| Section | Directory | Content |
|---------|-----------|---------|
| Architecture | `architecture/` | System-level design, layer diagrams, component interactions |
| Modules | `modules/` | Per-module documentation — one page per logical module or package |
| Decisions | `decisions/` | Architecture Decision Records (ADRs) — why a choice was made |
| Patterns | `patterns/` | Recurring design patterns used in the codebase |
| Concepts | `concepts/` | Domain concepts, glossary terms, mental models |
| Guides | `guides/` | How-to guides for specific tasks or workflows |
| Raw | `raw/` | Unprocessed inputs (meeting transcripts, notes). Not published. |

## Choosing a Section

- Changed how a module works? → `modules/`
- Made an architectural decision? → `decisions/`
- Noticed a recurring pattern? → `patterns/`
- Need to define a domain term? → `concepts/`
- Writing instructions for a task? → `guides/`
- Documenting system-level structure? → `architecture/`

## File Naming

- Use kebab-case: `auth-middleware.md`, `database-migration-strategy.md`
- Be descriptive: `auth.md` is less useful than `authentication-flow.md`
- Module pages should match their source path when possible: `internal/agent/` → `modules/agent.md`

## Navigation Files

These files are auto-generated by `plexium compile` — do not edit manually:

- `_index.md` — complete page listing with links
- `_Sidebar.md` — navigation sidebar
- `Home.md` — landing page

## Custom Sections

The `taxonomy.sections` config in `.plexium/config.yml` controls which sections exist. If you need a new section, add it there and create the directory.
